## 3. 各画面の詳細

### SCR-001 起動画面

**概要:** アプリ起動時に最初に表示される画面。DBファイルの場所を指定してログインする。

**UI要素:**

| 要素 | 種別 | 説明 |
|---|---|---|
| DBファイルパス表示欄 | テキスト表示 | 選択されたフォルダパスを表示。初回以降は前回パスを自動入力 |
| フォルダを選択ボタン | ボタン | フォルダ選択ダイアログを開く |
| パスワード入力欄 | パスワード入力 | manager認証用。未入力でログインするとmemberとして扱う |
| ログインボタン | ボタン | DBパスが指定されている場合のみ活性 |
| 新規作成ボタン | ボタン | SCR-002へ遷移 |
| エラーメッセージ表示欄 | テキスト | パスワード不一致・DBファイル未検出時に表示 |

**ビジネスロジック:**
- 前回開いたDBパスを `config.ini`（exeと同じディレクトリ）から読み込んでパス欄に自動入力する
- ログインボタン押下時:
  - パスワードあり: 入力値をSHA-256ハッシュ化してsettingsテーブルの値と比較。一致でmanagerログイン
  - パスワードなし: memberとしてログイン
  - DBファイルが存在しない場合はエラー表示
- 認証成功後、DBパスを `config.ini` に保存する

### SCR-002 初期設定画面

**概要:** 新規作成時のみ表示。DBファイルを作成しチームの基本設定を行う。

**UI要素:**

| 要素 | 種別 | 説明 |
|---|---|---|
| 保存先パス表示欄 | テキスト表示 | SCR-001で選択したフォルダパスを表示 |
| プレフィックス入力欄 | テキスト入力 | 必須。チケット番号のプレフィックス（例: ABC） |
| パスワード入力欄 | パスワード入力 | 必須。manager認証用パスワード |
| パスワード確認入力欄 | パスワード入力 | 必須。上記と一致するか確認 |
| 作成ボタン | ボタン | 全必須項目が入力済みの場合のみ活性 |
| キャンセルボタン | ボタン | SCR-001に戻る |

**ビジネスロジック:**
- 作成ボタン押下時:
  - バリデーション: プレフィックス・パスワード・パスワード確認が全て入力済みか
  - バリデーション: パスワードとパスワード確認が一致しているか
  - 指定フォルダに `rules.db`・`work.db` を作成する
  - パスワードをSHA-256ハッシュ化してsettingsテーブルに保存する
  - プレフィックスをsettingsテーブルに保存する
  - 完了後SCR-004へ遷移する

### SCR-003 カンバン設定画面

**概要:** manager限定。担当者・ステータス定義・タグ定義を管理する設定画面。別ウィンドウで表示。

**UI要素:**

**タブ①: 担当者管理**

| 追加ボタン | ボタン | 担当者追加フォームを表示 |
| 編集ボタン | ボタン | 選択した担当者の編集フォームを表示 |
| 無効化ボタン | ボタン | 選択した担当者をis_active=0にする（物理削除なし） |

**タブ②: ステータス管理**

| 要素 | 種別 | 説明 |
|---|---|---|
| ステータス一覧 | テーブル表示 | ステータス名・表示順を表示 |

### SCR-004 カンバンボード画面

**概要:** メイン画面。チケットをカンバン形式で表示する。

**UI要素:**

| 要素 | 種別 | 説明 |
|---|---|---|
| メニューバー | メニュー | カンバン設定・各機能への導線。manager限定項目はmemberには非表示 |
| フィルターエリア | FilterWidget（共通コンポーネント） | 後述のFilterWidget仕様を参照 |
| カンバンエリア | KanbanColumnWidget + KanbanCardWidget（共通コンポーネント） | 後述のカンバンエリア仕様を参照 |
| 新規チケットボタン | ボタン | manager限定。チケット作成画面へ |
| manager編集中警告バナー | テキスト表示 | 他のmanagerがロック中の場合に表示（memberのみ表示） |

**カンバンエリアの動作:**
- ステータス定義テーブルのdisplay_order順に列を左から右に生成する
- `default_hidden_status_ids` に登録されたステータスはデフォルトでフィルターから除外する
- チケットカードにはチケット番号・タイトル・担当者・終了予定日を表示する
- manager権限時: カードをドラッグ&ドロップで別の列に移動できる（ステータス変更）
- member権限時: ドラッグ&ドロップは無効。カードクリックで詳細表示のみ

## 4. 共通仕様

### FilterWidget（共通フィルターコンポーネント）

SCR-004・SCR-006・SCR-007で共通利用する再利用ウィジェット。

**UIレイアウト:**

```
┌─────────────────────────────────────────────────────┐
│ 担当者                                               │
│  [✓] 田中   [✓] 高橋   [✓] 鈴木                     │
│  ※複数選択時はOR条件（どれかが担当のチケットを表示）  │
│                                                     │
│ ステータス                                           │
│  [✓] 未着手  [✓] 仕掛  [✓] 完了  [ ] CLOSED     │
│  ※複数選択時はOR条件                                 │
│  ※default_hidden_status_idsのステータスは            │
│    起動時にチェックが外れた状態で初期化される          │
│                                                     │
│ タグ条件                        [+ 条件を追加]       │
│  [機種名 ▼] [含む ▼] [アルファ___________] [×]      │
│  [機能名 ▼] [含む ▼] [ビーム_____________] [×]      │
│  タグ条件間の結合: ( ● AND  ○ OR )                  │
│                                                     │
│ 開始日     [          ] 〜 [          ]             │
│ 終了予定日  [          ] 〜 [          ]             │
└─────────────────────────────────────────────────────┘
```

**各エリアの仕様:**

**担当者エリア:**
- 有効な担当者（is_active=1）を全員チェックボックスで表示する
- デフォルトは全員チェックON（全員表示）
- 複数選択時はOR条件（田中さんまたは高橋さんが担当のチケットを表示）

**ステータスエリア:**
- 全ステータスをチェックボックスで表示する
- デフォルトはsettingsの `default_hidden_status_ids` に登録されたステータスのみチェックOFF、それ以外はON
- 複数選択時はOR条件

**タグ条件エリア:**
- 「条件を追加」ボタンで行を追加する形式
- 各行の構成: `[タグ名ドロップダウン] [演算子ドロップダウン] [値テキスト入力] [×削除ボタン]`
- タグ名ドロップダウン: タグ定義一覧から選択
- 演算子ドロップダウン: 「含む（AND/OR）」「含まない（NOT）」から選択
- 行が複数ある場合、タグ条件間の結合方法をAND/ORでまとめて切り替える
- 「含まない」を選択した行はNOT条件として常に適用される（AND/ORの切り替えに関わらず）

**日付エリア:**
- 開始日・終了予定日それぞれにFROM/TOの日付入力欄を設ける
- 空欄の場合は制限なし

### カンバンエリア仕様

```
┌──────────┬──────────┬──────────┬──────────┐
│  未着手   │ 仕掛  │  完了    │  CLOSED  │
│          │          │          │          │
│ ┌──────┐ │ ┌──────┐ │ ┌──────┐ │          │
│ │ABC-1 │ │ │ABC-3 │ │ │ABC-5 │ │          │
│ │田中  │ │ │高橋  │ │ │田中  │ │          │
│ │〜2/28│ │ │〜3/10│ │ │〜2/20│ │          │
│ └──────┘ │ └──────┘ │ └──────┘ │          │
│ ┌──────┐ │          │          │          │
│ │ABC-2 │ │          │          │          │
│ │鈴木  │ │          │          │          │
│ │〜3/5 │ │          │          │          │
│ └──────┘ │          │          │          │
└──────────┴──────────┴──────────┴──────────┘
```

- 列はstatusesテーブルの `display_order` 昇順に左から右へ動的生成する
- 各カード（KanbanCardWidget）の表示項目: チケット番号・タイトル・担当者・終了予定日
- manager権限時: カードをドラッグ&ドロップで別の列に移動できる。移動完了時にステータスをDBに反映する
- member権限時: ドラッグ&ドロップは無効。カードクリックでSCR-005（詳細表示）へ遷移する

### タグエリア仕様（SCR-005 チケット詳細・編集画面）

```
┌─────────────────────────────┐
│ タグ                         │
│  機種名    [アルファ_______]  │
│  バージョン [1.0___________]  │
│  機能名    [ビーム発射______]  │
│  会社名    [________________] │
│  納期      [2026/03/31 📅  ] │  ← field_type=dateの場合は日付ピッカー
└─────────────────────────────┘
```

- タグ定義テーブルに登録されている全タグを定義順に縦に並べる
- field_type=text: テキスト入力欄
- field_type=date: 日付ピッカー
- 値が未入力のタグは保存時にスキップする（tag_valuesにレコードを作らない）
- manager権限時: 入力可能
- member権限時: 入力欄をdisableにして表示のみ

---

| 制御内容 | manager | member |
|---|---|---|
| チケット編集 | 可 | 不可（入力欄をdisable） |
| チケット削除 | 可 | 不可（ボタン非表示） |
| ドラッグ&ドロップ | 可 | 不可 |
| manager限定メニュー項目 | 表示 | 非表示 |
| manager編集中警告バナー | 非表示 | 表示（他managerがロック中の場合） |

